# Hazel Admin

꽃집 헤이즐을 위한 PWA 어드민 — 매출·지출·고객·예약·사진첩·인사이트를 한 앱에서 관리한다.

## 프로젝트 개요

Hazel Admin은 소규모 꽃집(헤이즐)의 일상 운영을 디지털화하는 PWA 어드민 웹앱이다. 매출 등록부터 카드 입금 대조, 고객 관리, 예약 리마인더, 인스타그램 인사이트까지 꽃집 사장이 매일 반복하는 업무를 단일 앱에서 처리한다. Next.js 16 App Router + Supabase 기반으로 구축되어 있으며, 멀티테넌시 아키텍처로 향후 여러 꽃집 계정을 지원할 수 있는 구조를 갖추고 있다. 현재는 단일 사용자(헤이즐 사장) 운영이 주목적이며, 공개 홈페이지 추가를 통해 고객 접점도 확장 중이다.

## 타겟 사용자

| 사용자 | 역할 | 접근 경로 |
|--------|------|-----------|
| 꽃집 헤이즐 사장님 (1인) | 모든 어드민 기능 사용 — 매출 입력, 고객 관리, 예약, 설정 | `/admin/*` (로그인 필요) |
| 홈페이지 방문 고객 (향후) | 꽃집 소개, 컬렉션 확인, 주문 문의 | `/(public)` (인증 불필요) |

## 핵심 기능

### 1. 대시보드
- 월별 매출 합계·주요 지표 요약 카드
- 실시간 집계 (DB 하드코딩 없음)

### 2. 매출 관리
- 매출 등록 (결제 수단, 채널, 금액, 고객 연결, 사진 첨부)
- 로드 구입 간편 모드 (현금·road 고정)
- 미수(외상) 처리: `completeUnpaidSale()` / `revertUnpaidSale()`
- CSV/Excel/PDF 내보내기

### 3. 지출 관리
- 지출 항목 등록 (단가 × 수량)
- 카테고리별 분류 및 목록 관리

### 4. 고객 관리
- 전화번호 기준 고객 식별 (user_id 복합 unique)
- 성별 선택, 고객 상세 다이얼로그, 자동완성

### 5. 예약 캘린더
- 예약 등록·수정·삭제
- `reminder_at` 기반 푸시 알림 발송
- 캘린더에서 직접 사진 첨부·삭제

### 6. 사진첩
- Cloudflare R2 저장, 3MB 초과 자동 압축
- 카드당 최대 10장, CDN 캐싱

### 7. 입금 대조
- 카드사별 수수료율 적용 (`expected_deposit = amount × (1 - fee_rate/100)`)
- 영업일 기준 입금 예정일 계산

### 8. 인사이트
- **트렌드**: RSS 수집 아티클, 카테고리 필터, 스크랩 토글
- **팔로우**: 인스타그램 피드, 계정별 필터, 라이트박스 뷰
- **스크랩**: 북마크 모음 + 메모 편집(포커스 해제 시 자동 저장)

### 9. 설정
- 카드사 추가·수정, 푸시 알림 구독 관리
- BottomNav 항목 커스텀 (4~6개, drag-and-drop 정렬)

## 사용 시나리오

### 시나리오 1: 일별 매출 입력
사장이 하루 영업 마감 후 앱을 열고 매출 탭으로 이동한다. "+" 버튼을 눌러 결제 수단(현금/카드), 채널(주문방식), 금액, 고객명을 입력한다. 사진이 있으면 최대 10장 첨부한다. 저장하면 대시보드 집계가 즉시 갱신된다.

### 시나리오 2: 예약 → 매출 전환 + 리마인더
웨딩 예약을 캘린더에 등록하고 `reminder_at`을 하루 전 오전으로 설정한다. 당일 Vercel Cron이 푸시 알림을 발송하면 Service Worker가 알림을 표시한다. 행사 후 예약을 매출로 전환하면서 사진과 금액을 함께 기록한다.

### 시나리오 3: 인사이트 스크랩
인사이트 → 팔로우 탭에서 팔로우 중인 플로리스트의 최신 피드를 확인한다. 마음에 드는 포스트에 북마크를 토글해 스크랩하고, 상세 다이얼로그에서 메모를 작성한다. 스크랩 탭에서 트렌드 아티클과 함께 모아 볼 수 있다.

## 비기능 요구사항

| 요구사항 | 내용 |
|----------|------|
| 멀티테넌시 | 12개 테이블 `user_id` 컬럼 + RLS `auth.uid() = user_id` — DB 레벨 격리 |
| RLS 격리 | 인증 누락 시에도 RLS가 2차 방어선 역할. user_id 삽입 누락 = 보안 결함 |
| 한국어 UI | 라벨·메시지·플레이스홀더 전부 한국어 |
| PWA + 푸시 | Service Worker + VAPID Web Push, 홈 화면 설치 지원 |
| 모바일 우선 | BottomNav(모바일·태블릿) / Sidebar(데스크톱), lg 브레이크포인트 기준 |
| 다크모드 | next-themes + CSS 변수 (`--brand`, `--sage` 등), 하드코딩 색상 금지 |
| 개인정보 | 고객 전화·이름은 user_id 스코프 격리 필수. 외부 노출 금지 |
| 접근성 | 아이콘 버튼 `aria-label`, 클릭 카드 `role="button"` + 키보드 이벤트 |

## 비목표

다음 항목은 현재 명시적 out-of-scope이다.

- **대규모 트래픽 확장**: 수천 명의 동시 사용자 처리 — 소규모 단독 운영이 목적
- **외부 결제 통합**: PG사 연동, 온라인 결제 — 오프라인 매출 기록이 목적
- **다국어 지원**: 한국어 단일 언어 — 국제화(i18n) 계획 없음
- **다중 매장 관리**: 지점별 분리 운영 — 단일 꽃집 운영 도구
- **재고 관리**: 꽃 재고·발주 시스템 — 범위 외
- **고객용 주문 플랫폼**: 전자상거래 — 관리자 도구가 목적 (홈페이지는 소개·문의 수준)

## 현재 진행

브랜치 `feature/init-website`에서 다음 두 가지가 동시 진행 중이다.

**라우팅 마이그레이션**
- `src/app/(dashboard)/*` → `src/app/(admin)/admin/*` (36개 파일 staged)

- `src/app/(public)/` + `src/components/public/` 신규 생성
- 8개 섹션: hero / about / collection / instagram / location / order / header / footer
