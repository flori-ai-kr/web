# 멀티 픽업 예약 설계

> 하나의 매출(결제)에서 여러 예약(픽업)이 파생될 수 있도록 변경

## 핵심 개념

- **매출(Sale)** = 결제 단위: 날짜, 총액, 결제방식, 고객, 카드사
- **예약(Reservation)** = 픽업 단위: 픽업 날짜, 시간, 제목, 예상 금액, 제작/픽업 상태
- **관계**: 1 매출 : N 예약 (`reservations.sale_id` → `sales.id`)

## DB 변경

### 마이그레이션

1. `sales.reservation_id` 컬럼 삭제 (기존 1:1 제약 제거)
   - 기존 연결 정보는 `reservations.sale_id`에 이미 존재하므로 데이터 손실 없음
2. `reservations.sale_id`는 유지 (N:1 FK)
3. `reservations.payment_date` 컬럼은 더 이상 필요 없음 → 삭제 고려
   - 매출 날짜가 결제일이 되므로 별도 필드 불필요

### 타입 변경

- `Sale` 타입에서 `reservation_id` 제거
- `Reservation` 타입에서 `payment_date` 제거

## 캘린더 예약 생성 플로우

### 첫 예약 생성 (기존과 유사)

1. 사용자가 캘린더에서 날짜 클릭 → "추가" 버튼
2. 폼 입력:
   - 제목, 고객명, 전화번호, 픽업 시간, 예상 금액
   - 상품 카테고리, 결제방식, 예약 채널
   - **"결제일자 지정" → 제거** (매출 날짜 = 선택한 날짜 기준)
     - 대신 **"픽업일자 지정"** 체크박스: 체크 시 픽업 날짜를 별도 지정 가능
     - 체크 안 하면 픽업 날짜 = 선택한 날짜 (현재 동작과 동일)
3. 제출 시:
   - 매출 생성 (date = 선택한 날짜)
   - 예약 생성 (date = 픽업일자, sale_id = 생성된 매출 ID)

### 픽업 추가 (새 기능)

1. 캘린더 예약 상세 패널에서 **"픽업 추가"** 버튼
2. 간소화된 폼:
   - 픽업 날짜, 시간, 제목, 예상 금액
   - (고객 정보, 결제 정보는 매출에서 상속 → 입력 불필요)
3. 같은 `sale_id`로 새 예약 생성

## UI 변경

### 캘린더 예약 상세 패널

- 같은 `sale_id`를 가진 예약들을 **그룹으로 표시**
- 그룹 헤더: 고객명, 총 매출 금액, "매출 확인" 링크
- 각 픽업: 날짜, 시간, 제목, 예상 금액, 제작완료/픽업완료 토글
- "픽업 추가" 버튼 (같은 매출에 예약 추가)

### 캘린더 월간/5일 뷰

- 각 예약은 해당 **픽업 날짜**에 독립 표시 (변경 없음)
- 같은 매출의 다른 예약 날짜가 있으면 카드에 표시 (예: "2/9, 2/20")

### 캘린더 예약 폼

- "결제일자 지정" → **"픽업일자 지정"**으로 레이블 변경
- 체크 시: 픽업 날짜 입력 필드 표시 (매출 날짜와 다른 날짜에 픽업)
- 체크 안 하면: 픽업 날짜 = 현재 선택 날짜

### 매출 상세 다이얼로그 (SaleDetailDialog)

- 연결된 예약 목록 섹션 추가
- 각 예약: 픽업 날짜, 시간, 제목, 상태 표시

## Server Action 변경

### reservations.ts

- `createReservation`: 변경 없음
- `convertReservationToSale`: 매출 생성 시 `reservation_id` 설정 제거
- **새 함수 `addPickupToSale`**: 기존 매출에 예약 추가
  - 매출 ID + 픽업 폼 데이터 → 새 예약 생성 (sale_id 연결)
  - 고객 정보는 매출에서 가져옴

### sales.ts

- `createSale`: `reservation_id` 파라미터 제거
- **새 함수 `getReservationsForSale`**: 매출에 연결된 예약 목록 조회

## 금액 처리

- 매출 금액은 처음 등록 시 한 번만 설정 (변경 없음)
- 각 예약의 예상 금액은 참고용 표시만
- 합산/동기화 없음 (단순함 유지)
